### Connection Pattern Analysis

```typescript
// Key implementation in src/lib/db/index.ts:
// 1. Lazy initialization prevents build-time errors
let _db: NeonHttpDatabase<typeof schema> | null = null;

// 2. Validation before connection
const isDatabaseConfigured = () => {
  const url = process.env.DATABASE_URL;
  return !!url && url !== "postgresql://placeholder";
};

// 3. Proxy pattern for lazy access
export const db = new Proxy({} as NeonHttpDatabase<typeof schema>, {
  get(_target, prop) {
    const database = getDatabase();
    return /* ... */;
  },
});
```

### Architecture Observations

1. **Organization Isolation**: All brand-related queries filter by `user.orgId` for multi-tenant security
2. **Computed Values**: GEOScore is calculated on-the-fly from mentions/recommendations rather than stored
3. **Denormalized Fields**: GraphQL type field transformations (e.g., `monitoringPlatforms` → `platforms`)
