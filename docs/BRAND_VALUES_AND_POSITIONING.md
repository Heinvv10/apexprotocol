## White-Label Architecture Strategy

### Core Principle: Modular Brand Identity System

**CRITICAL REQUIREMENT**: The UI/UX must be modular to support easy white-labeling while maintaining brand value integrity.

### What Changes vs. What Stays

#### 🔒 IMMUTABLE (Core Brand Values - Cannot Change)
These elements define the platform's credibility and must remain consistent:
- **Trust signals**: Verification badge system, audit icons, source transparency
- **Excellence dimensions**: Impact Index, Innovation Score, Influence Score calculations
- **Quality standards**: Data accuracy, methodology transparency
- **Positioning principles**: Private Club exclusivity, Business Oracle insights, Credibility Engine verification

**Why**: These are what make the platform valuable. Removing them destroys the "Forbes+Gartner+BlackRock" positioning.

#### 🎨 CUSTOMIZABLE (Visual Brand Identity - White-Label Ready)
These elements can be customized per client without losing platform integrity:
- **Logo and brand name**: Company wordmark, favicon, loading animations
- **Color palette**: Primary, secondary, accent colors (while maintaining contrast ratios)
- **Typography**: Font families (Inter can be replaced with brand fonts)
- **Spacing/sizing**: Border radius, padding scale (within design system constraints)
- **Copywriting**: Headlines, taglines, messaging tone
- **Illustrations**: Custom graphics, photography style, iconography

**Why**: Visual identity customization enables white-labeling while preserving functional trust signals.

#### Layer 2: CSS Custom Properties (Theme Tokens) 🎨

**Theme Provider System** (`app/providers/theme-provider.tsx`):
```typescript
'use client';

import { createContext, useContext, useEffect, useState } from 'react';

type Theme = 'light' | 'dark' | 'system';

type ThemeConfig = {
  brandName: string;
  primaryColor: string;
  logo: string;
  // ... all brand tokens
};

const ThemeContext = createContext<{
  theme: Theme;
  setTheme: (theme: Theme) => void;
  brandConfig: ThemeConfig;
}>({
  theme: 'system',
  setTheme: () => {},
  brandConfig: {} as ThemeConfig,
});

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const brandConfig: ThemeConfig = {
    brandName: process.env.NEXT_PUBLIC_BRAND_NAME || 'Apex',
    primaryColor: process.env.NEXT_PUBLIC_PRIMARY_COLOR || '#4926FA',
    logo: process.env.NEXT_PUBLIC_BRAND_LOGO_URL || '/logos/default.svg',
    // Load all theme tokens from env vars
  };

  // Apply CSS custom properties
  useEffect(() => {
    const root = document.documentElement;
    root.style.setProperty('--primary', brandConfig.primaryColor);
    root.style.setProperty('--brand-name', brandConfig.brandName);
    // ... set all CSS variables
  }, [brandConfig]);

  return (
    <ThemeContext.Provider value={{ theme, setTheme, brandConfig }}>
      {children}
    </ThemeContext.Provider>
  );
}
```
