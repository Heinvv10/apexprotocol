## Critical Issues (Fix Now)

### 1. XSS — Unsanitized AI HTML in Content Preview (WEEK 2 — UNRESOLVED)
**File:** `src/components/create/content-preview-modal.tsx:83`
**Status:** First flagged 2026-04-26. Still unresolved.

AI-generated HTML is passed directly to `dangerouslySetInnerHTML` with no sanitization. A poisoned AI response or stored XSS payload would execute in the user's browser.

```tsx
// CURRENT — unsafe
dangerouslySetInnerHTML={{ __html: content || "<p>...</p>" }}

// FIX — add DOMPurify
import DOMPurify from "isomorphic-dompurify";
dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(content) }}
```

**Fix:** `npm install isomorphic-dompurify` (SSR-safe), wrap the one line.

### 2. Missing Production Dependency — `@tanstack/react-query` (WEEK 2 — UNRESOLVED)
**File:** `package.json`

### 3. 250 Production Files Use `console.error`/`console.warn` Instead of Structured Logger
**Files:** 250 production `.ts`/`.tsx` files (568 total occurrences)
**Examples:**
- `src/components/create/rich-text-editor-core.tsx:353,368,377,392` — 4 console.error calls
- `src/components/features/content/GenerateContentForm.tsx:177,192` — console.warn + console.error
- `src/components/audit/ScheduleModal.tsx:69` — `console.error("Error:", error)` (no context)
- `src/components/optimization/suggestion-panel.tsx:92` — `console.error("Failed to copy:", err)`

**Fix priority:** Error boundaries first (`error-boundary.tsx:34,224,373`), then data-fetch catch blocks.

### 4. `/api/export/history` Has No Route-Level Auth Check
**File:** `src/app/api/export/history/route.ts:14`

The route returns export job history from an in-memory Map. While `/api/export(.*)` is listed in `apiKeyAuthRoutes` in middleware, the route handler itself performs zero auth validation — any valid API key can read any user's export history. The in-memory store is shared across all requests.

### 6. Rate Limiter Fails Open Silently (Unchanged from Last Week)
**File:** `src/middleware.ts:142,158`

When Upstash/Redis is unavailable, catch blocks silently allow all traffic through with no logging. A Redis outage disables rate limiting with no alerting.

```ts
} catch {
  // rate limiter unavailable — fail open (no logging)
}
// Fix:
} catch (err) {
  logger.warn("rate-limiter unavailable, failing open", { err });
}
```

### 15. `@tanstack/react-query-devtools` in `devDependencies` But No CI Gate
**File:** `package.json`

The devtools package transitively provides `@tanstack/react-query` today. Without a `npm ci --omit=dev` step in CI, this fragile dependency path goes untested. Add a CI step that runs `npm ci --omit=dev && npm run build` to catch this class of missing-prod-dep errors.

---

### 16. No HTML Sanitization Audit for TipTap Editor Output
`@tiptap/react` stores HTML output. Audit all paths that render TipTap-produced HTML — any `dangerouslySetInnerHTML` rendering of stored TipTap content is a stored XSS vector. Current known risk: `content-preview-modal.tsx:83`.

---

### 17. `ESLint @typescript-eslint/no-explicit-any` is Warning Not Error
**File:** `eslint.config.mjs:31`

The ESLint comment notes `--max-warnings=0` on new PRs theoretically catches new `any` introductions. However, the rule is `"warn"` not `"error"` — if `--max-warnings=0` is not consistently enforced in all CI pipelines, new `any` usages will silently accumulate. Raise to `"error"` for the `src/` directory and exclude only the known legacy debt via `overrides`.
