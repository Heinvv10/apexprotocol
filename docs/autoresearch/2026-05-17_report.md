## Critical Issues (Fix Now)

1. **XSS — unsanitized HTML rendered directly** — `src/components/create/content-preview-modal.tsx:83` — AI-generated content injected via `dangerouslySetInnerHTML={{ __html: content }}` with no sanitization; malicious content from the AI or a compromised response can execute arbitrary JS in the user's browser. **Fix:** `npm i dompurify @types/dompurify isomorphic-dompurify` and wrap: `__html: DOMPurify.sanitize(content || "")`. *Unresolved for 3 consecutive weeks.*

2. **Missing prod dependency — app crashes in production** — `package.json` — `@tanstack/react-query` is used in 476 places across `src/` but only `@tanstack/react-query-devtools` appears in `devDependencies`. Production builds strip devDependencies; any serverless deployment without a pre-bundled node_modules will fail at runtime. **Fix:** `npm i @tanstack/react-query`. *Unresolved for 3 consecutive weeks.*

3. **`any` type count regressed 178 → 266** — codebase-wide — The gains from last week (+34%) have been reversed. New `any` usages appeared across `simulate/`, `brands/completion-wizard`, `forms/form-wrapper`, and `audit/` components. With `strict: true` in `tsconfig.json`, these represent active suppressions of the type system. **Fix:** Enforce `@typescript-eslint/no-explicit-any: error` in ESLint config and ban new merges that introduce `any`.

4. **`/api/generate` has no user authentication** — `src/app/api/generate/route.ts` — The content generation endpoint accepts arbitrary prompts and calls Anthropic/OpenAI at org cost with no session or API key check. In `devMiddleware` (when `SUPABASE_AUTH_CONFIGURED=false`) the request passes straight through. Even in production middleware it falls outside both `publicRoutes` and `apiKeyAuthRoutes`, meaning it reaches Supabase session check — but dev mode bypasses this entirely. Any unauthenticated actor can burn AI credits. **Fix:** Add `const session = await getServerSession()` at the top and return 401 if absent, or add `/api/generate(.*)` to `apiKeyAuthRoutes` in `middleware.ts`.

5. **Export jobs stored in in-memory `Map` — data lost on cold starts** — `src/app/api/export/route.ts:42` — `const exportJobs = new Map<string, ExportJob>()` is module-level state. In serverless/edge deployments each cold start creates a fresh map; any job created on instance A is invisible to instance B. The GET `/api/export/history` route on line 1147 reads only from this map, so users see empty history after any restart. **Fix:** Persist export jobs to Redis (`@upstash/redis` is already a dep) or to the PostgreSQL `exports` table if one exists.

## Warnings (Fix Soon)

1. **576 `console.error`/`console.warn` calls in prod code** — codebase-wide — The project has a structured logger at `src/lib/logger.ts` with explicit instruction: *"Replaces ad-hoc `console.log`. Do not introduce new `console.*` calls in src/."* 576 calls remain unported. **Suggested fix:** Add `no-console` ESLint rule; fix in batches by module.

2. **Docker-compose references `CLERK_*` env vars post-migration** — `docker-compose.yml:30-32` — `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`, `CLERK_SECRET_KEY`, `CLERK_WEBHOOK_SECRET` are still wired into the container environment after the Clerk → Supabase migration (`78c8053`). These will never be set in new deployments and may cause confusion during incident debugging. Remove the dead vars; add the missing Supabase equivalents if not already there.

3. **Dual Google AI SDKs** — `package.json:27-28` — Both `@google/genai@^1.34.0` (new unified SDK) and `@google/generative-ai@^0.21.0` (old deprecated SDK) are listed. This doubles bundle size for that dependency tree and creates confusion about which API surface to use. **Fix:** Audit all imports, migrate to `@google/genai`, remove `@google/generative-ai`.

4. **SWR and TanStack Query both active** — 60 SWR usages vs 476 TanStack Query usages — Two overlapping server-state libraries are maintained in parallel. SWR adds ~7 KB gzipped bundle cost with no unique benefit over TanStack Query here. **Fix:** Migrate 60 SWR calls to `useQuery`/`useMutation` and remove the `swr` dependency.

5. **0 React.memo usages across 44 component directories** — `src/components/` — The dashboard renders complex charts (Recharts) and large data lists. Without memoization, any parent re-render cascades to all children. With 96 `useMemo`/`useCallback` hooks providing stable references but no `memo()` to short-circuit, those stabilized props have no consumer to benefit from them. **Fix:** Wrap frequently-rendered pure components (chart wrappers, list items, sidebar nav items) with `memo()`.

6. **4 raw `<img>` tags not using `next/image`** — `src/app/report/[brandId]/[period]/page.tsx:160,169`, `src/app/dashboard/brands/new/new-brand-client.tsx:515`, `src/app/dashboard/brands/[id]/edit/page.tsx:494` — Missing lazy-loading, automatic WebP conversion, and layout-shift prevention. *Unresolved for 3 consecutive weeks.*

7. **Rate limiter fails open silently** — `src/middleware.ts:251` — The `} catch { // rate limiter unavailable — fail open }` block swallows all Redis errors without logging. If Upstash is down, every user gets unlimited API calls with no visibility into the failure. **Fix:** Log the error via `logger.warn` at minimum before letting through.

8. **`graphql/schema.ts` at 2,284 lines** — `src/lib/graphql/schema.ts` — Single-file GraphQL schema is unmaintainable at this size. Resolvers, type definitions, and context setup are all co-located. **Fix:** Split into `schema/types/*.ts`, `schema/resolvers/*.ts`, and `schema/context.ts`.

9. **`/api/marketing/*` routes not covered by API key auth** — `src/app/api/marketing/` — Marketing routes (`/overview`, `/emails`, `/campaigns`, `/calendar`, `/analytics`, `/sequences`, `/templates`) are absent from both `publicRoutes` and `apiKeyAuthRoutes` in middleware. In production they fall through to Supabase session check (OK), but in dev mode (`SUPABASE_AUTH_CONFIGURED=false`) they pass unauthenticated. **Fix:** Add `/api/marketing(.*)` to `apiKeyAuthRoutes`.
