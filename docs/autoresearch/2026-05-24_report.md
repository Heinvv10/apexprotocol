## Critical Issues (Fix Now)

1. **XSS ‚Äî unsanitized HTML rendered via `dangerouslySetInnerHTML`** ‚Äî `src/components/create/content-preview-modal.tsx:83` ‚Äî AI-generated content passed directly as `__html: content` with no sanitization. Malicious content from a compromised AI response can execute arbitrary JS in the user's browser. **Fix:** `npm i isomorphic-dompurify @types/dompurify` then wrap: `__html: DOMPurify.sanitize(content ?? "")`. *Unresolved 4 consecutive weeks.* (Note: `faq-section.tsx:109` uses `dangerouslySetInnerHTML` too but only for structured JSON-LD schema ‚Äî that usage is safe as it serializes a known JS object, not user input.)

2. **Missing production dependency ‚Äî will crash in serverless deployments** ‚Äî `package.json` ‚Äî `@tanstack/react-query` is imported in 54 source files but only `@tanstack/react-query-devtools` is in `devDependencies`. Production Docker builds strip devDependencies; any deployment without pre-bundled node_modules silently fails at runtime. **Fix:** `npm i @tanstack/react-query`. *Unresolved 4 consecutive weeks.*

3. **In-memory export jobs lost on every serverless restart** ‚Äî `src/app/api/export/route.ts:42` ‚Äî `const exportJobs = new Map<string, ExportJob>()` is module-level state. Cold starts create a fresh map; instance A's jobs are invisible to instance B. The GET `/api/export/history` route reads only from this map, so users see empty history after any restart. **Fix:** Move job state to `@upstash/redis` (already installed) with a TTL-keyed hash. *Unresolved 3 consecutive weeks.*

4. **`/api/generate` has no authentication** ‚Äî `src/app/api/generate/route.ts:6` ‚Äî No session, API key, or user check before calling Anthropic/OpenAI at org cost. Any unauthenticated actor can burn AI credits. In dev mode (`SUPABASE_AUTH_CONFIGURED=false`) the middleware passes the route through entirely. **Fix:** Add `/api/generate(.*)` to `apiKeyAuthRoutes` in `src/middleware.ts:80` or add `const { data: { user } } = await supabase.auth.getUser()` at the top of the handler. *Unresolved 3 consecutive weeks.*

5. **5 additional API routes with no authentication** ‚Äî newly identified ‚Äî The following routes have no `createServerClient`, `getUser`, or session check and are not in the public/cron allowlists:
   - `src/app/api/jobs/route.ts` ‚Äî exposes full BullMQ queue state (job details, payloads)
   - `src/app/api/gamification/route.ts` ‚Äî reads/writes user XP and achievements
   - `src/app/api/graphql/route.ts` ‚Äî full GraphQL schema + queries, 2,284-line schema exposed unauthenticated
   - `src/app/api/admin/fix-linkedin-url/route.ts` ‚Äî admin mutation, no auth
   - `src/app/api/admin/seed-platform/route.ts` ‚Äî admin data seeding, no auth
   - **Fix:** Each should either check the Supabase session at handler entry or have its path added to the `apiKeyAuthRoutes` array in `middleware.ts`.
