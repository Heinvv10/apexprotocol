## Critical Issues (Fix Now)

1. **XSS — unsanitized HTML rendered via `dangerouslySetInnerHTML`** — `src/components/create/content-preview-modal.tsx:83` — `__html: content` receives raw AI-generated text with no sanitization. A compromised AI response or SSRF vector can execute arbitrary JS in the user's session. *Unresolved 5 consecutive weeks.*
   **Fix:** `npm i isomorphic-dompurify @types/dompurify`, then:
   ```tsx
   dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(content ?? "") }}
   ```

2. **Missing production dependency — runtime crash on cold starts** — `package.json` — `@tanstack/react-query` is imported in **52 source files** but exists only in `devDependencies`. Docker `npm ci --omit=dev` strips it; any serverless/Docker deployment without pre-bundled `node_modules` crashes at runtime. *Unresolved 5 consecutive weeks.*
   **Fix:** `npm i @tanstack/react-query`

3. **In-memory export job state lost on every restart** — `src/app/api/export/route.ts:42` — `const exportJobs = new Map<string, ExportJob>()` is module-level state. Restarts, scaled instances, and cold starts create a fresh map — users see empty export history. Comment on line 42 even acknowledges: *"In production, use Redis or database"*.
   **Fix:** Move to `@upstash/redis` with TTL hash. *Unresolved 4 weeks.*

4. **`/api/generate` — unauthenticated AI cost exposure** — `src/app/api/generate/route.ts` — No session check, no API key validation. Any actor can POST to this endpoint and burn Anthropic/OpenAI credits at org expense. When `SUPABASE_AUTH_CONFIGURED=false` (dev fallback), middleware skips this route entirely. *Unresolved 4 weeks.*
   **Fix:** Add auth guard at handler entry:
   ```ts
   const { data: { user } } = await supabase.auth.getUser();
   if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
   ```

5. **`/api/jobs` — BullMQ internals exposed unauthenticated** — `src/app/api/jobs/route.ts` — 335-line route exposes full BullMQ queue state, job payloads, and job management (including cancel/retry) with no `createServerClient`, `getUser`, or session check found in the file. Job payloads may contain user data, API keys, or org identifiers.
   **Fix:** Add `requireAuthContext` from `@/lib/auth/api-key-context` at the top of each handler.

6. **`/api/gamification` — XP and achievement data writeable without auth** — `src/app/api/gamification/route.ts` — 479-line route reads/writes `userGamification` and `userAchievements` DB tables. No auth guard found. Any actor can POST to inflate XP, award achievements, or read leaderboard data for any userId.
   **Fix:** Same as above — add session check before any DB operation.
