## Critical Issues (Fix Now)

1. **XSS — unsanitized HTML via `dangerouslySetInnerHTML`** — `src/components/create/content-preview-modal.tsx:83` — `__html: content` receives raw AI-generated text with no sanitization. *Unresolved 6 consecutive weeks.*
   **Fix:** `npm i isomorphic-dompurify @types/dompurify` then:
   ```tsx
   dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(content ?? "") }}
   ```

2. **Missing production dependency — 52 files import `@tanstack/react-query` which is absent from `package.json`** — `package.json` — Library is listed in neither `dependencies` nor `devDependencies`. Docker bun builds and any `npm ci --omit=dev` deployment will fail at runtime. *Unresolved 6 consecutive weeks.*
   **Fix:** `bun add @tanstack/react-query`

3. **In-memory export job state lost on every restart** — `src/app/api/export/route.ts:42` — `const exportJobs = new Map<string, ExportJob>()` is module-level state. Comments acknowledge: *"In production, use Redis or database"*. *Unresolved 5 weeks.*
   **Fix:** Move to `@upstash/redis` with TTL hash.

4. **`/api/generate` — unauthenticated AI cost exposure** — `src/app/api/generate/route.ts:5` — No session check. Any actor can POST and burn Anthropic/OpenAI credits. *Unresolved 5 consecutive weeks.*
   **Fix:** Add at top of POST handler:
   ```ts
   const { data: { user } } = await supabase.auth.getUser();
   if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
   ```

5. **`/api/jobs` — BullMQ internals exposed unauthenticated** — `src/app/api/jobs/route.ts` — 335-line route exposes full queue state, job payloads, job management (cancel/retry). No auth keyword found in file. *Unresolved 5 weeks.*

6. **`/api/gamification` family — XP, achievements, badges, challenges all writeable without auth** — `src/app/api/gamification/route.ts` (479 lines), `src/app/api/gamification/badges/route.ts`, `src/app/api/gamification/challenges/route.ts` — Zero auth checks in all three. Any actor can inflate XP, award achievements, or manipulate challenge state for any userId. Badges and challenges are newly confirmed this week.

7. **`/api/export` family — export history and file downloads accessible without auth** — `src/app/api/export/history/route.ts`, `src/app/api/export/[id]/route.ts`, `src/app/api/export/[id]/download/route.ts`, `src/app/api/export/templates/route.ts` — All read from the in-memory `exportJobs` Map imported from `../route`. No auth in any of these four files. An actor with a known job UUID can download another user's export.
   **Fix:** Add `getOrganizationId` guard at handler entry for all four routes.

8. **`CLERK_CONFIGURED` dead reference in live code path** — `src/lib/auth/api-key-context.ts:37-39` — Post-Supabase migration, code still checks `process.env.NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`. In development when this env var is absent, the guard at line 144 bypasses auth entirely (`if (!CLERK_CONFIGURED && process.env.NODE_ENV === 'development')`). Any dev-mode deployment with this var absent is fully unauthenticated.
