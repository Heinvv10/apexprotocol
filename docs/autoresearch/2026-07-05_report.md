## Critical Issues (Fix Now)

All 8 issues below are carried over from June 28. None were resolved. Each is now 9+ weeks old.

1. **XSS — unsanitized HTML via `dangerouslySetInnerHTML`** — `src/components/create/content-preview-modal.tsx:83` — `__html: content` receives raw AI-generated text with zero sanitization. Any prompt-injected `<script>` or `<img onerror=...>` payload executes in the user's browser. *Unresolved for 8 consecutive weeks.*
   **Fix:** `npm i isomorphic-dompurify @types/dompurify` then:
   ```tsx
   dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(content ?? "") }}
   ```

2. **Missing production dependency — 52 files import `@tanstack/react-query` which is absent from `package.json`** — `package.json:100` — Library is listed only as `@tanstack/react-query-devtools` in `devDependencies`, not the actual runtime library. Any `npm ci --omit=dev` / Docker production build will fail at runtime. *Unresolved for 8 consecutive weeks.*
   **Fix:** `npm i @tanstack/react-query`

3. **In-memory export job state lost on every restart + no auth on history** — `src/app/api/export/route.ts:87` — `const exportJobs = new Map<string, ExportJob>()` is module-level state (confirmed: no auth in POST at line 87 or GET at line 1144). `src/app/api/export/history/route.ts` imports this Map directly and has **no auth check** — any caller can enumerate export history of all users via GET. *Unresolved for 7 weeks.*
   **Fix:** Move to `@upstash/redis` with TTL hash; add `requireAuth` guard to both POST and GET handlers.

4. **`/api/generate` — unauthenticated AI cost exposure** — `src/app/api/generate/route.ts:6` — No session check. Any external actor can POST to this endpoint and consume Anthropic/OpenAI credits. Route is not in `publicRoutes` in middleware, suggesting the omission is a bug. *Unresolved for 7 consecutive weeks.*
   **Fix:** Add at top of POST handler:
   ```ts
   const { data: { user } } = await supabase.auth.getUser();
   if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
   ```

5. **`/api/jobs` — BullMQ internals exposed unauthenticated** — `src/app/api/jobs/route.ts` — 335-line route exposes full queue state, job payloads, cancel/retry management. No auth imports in file. Anyone can enumerate job payloads or cancel production jobs. *Unresolved for 7 weeks.*

6. **`/api/gamification` family — XP, achievements, badges, challenges all writeable without auth** — `src/app/api/gamification/route.ts` (479 lines), `src/app/api/gamification/badges/route.ts`, `src/app/api/gamification/challenges/route.ts` — Zero auth checks in all three files. Any unauthenticated actor can inflate XP, award achievements, or modify challenge state for any `userId` supplied in the request body. *Unresolved for 7 weeks.*

7. **`/api/export` family — export history and downloads accessible without auth** — `src/app/api/export/history/route.ts`, `src/app/api/export/[id]/route.ts`, `src/app/api/export/[id]/download/route.ts`, `src/app/api/export/templates/route.ts` — No auth in any of these four files. A caller with a known job UUID can download another user's export file. *Unresolved for 7 weeks.*

8. **`CLERK_CONFIGURED` dead reference bypasses auth in development** — `src/lib/auth/api-key-context.ts:37-39` — Post-Supabase migration, code still checks `process.env.NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`. When that var is absent (expected in new dev setups), the guard at line 144 bypasses auth entirely with `if (!CLERK_CONFIGURED && process.env.NODE_ENV === 'development')`. Any dev deployment without this legacy var is fully unauthenticated. *Unresolved for 7 weeks.*
