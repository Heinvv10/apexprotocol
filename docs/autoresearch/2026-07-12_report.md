## Critical Issues (Fix Now)

All 8 issues below are carried over from July 5. None were resolved.

1. **XSS ‚Äî unsanitized HTML via `dangerouslySetInnerHTML`** ‚Äî `src/components/create/content-preview-modal.tsx:83` ‚Äî `__html: content` receives raw AI-generated text with zero sanitization. Any prompt-injected `<script>` or `<img onerror=...>` payload executes in the user's browser. *Unresolved for 9 consecutive weeks.*
   **Fix:** `npm i isomorphic-dompurify @types/dompurify` then:
   ```tsx
   dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(content ?? "") }}
   ```

2. **Missing production dependency ‚Äî 52 files import `@tanstack/react-query` which is absent from `package.json`** ‚Äî `package.json` (devDependencies only has `@tanstack/react-query-devtools`) ‚Äî Any `npm ci --omit=dev` / Docker production build will fail at runtime. *Unresolved for 9 consecutive weeks.*
   **Fix:** `npm i @tanstack/react-query`

3. **In-memory export job state lost on every restart + no auth on history** ‚Äî `src/app/api/export/route.ts:42` ‚Äî `const exportJobs = new Map<string, ExportJob>()` is module-level state. `src/app/api/export/history/route.ts` has no auth check ‚Äî any caller can enumerate all export history. *Unresolved for 8 weeks.*
   **Fix:** Move to `@upstash/redis` with TTL hash; add `requireAuth` guard to POST and GET handlers.

4. **`/api/generate` ‚Äî unauthenticated AI cost exposure** ‚Äî `src/app/api/generate/route.ts` ‚Äî No session check. Any external actor can POST to this endpoint and consume Anthropic/OpenAI credits. *Unresolved for 8 consecutive weeks.*
   **Fix:**
   ```ts
   const { data: { user } } = await supabase.auth.getUser();
   if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
   ```

5. **`/api/jobs` ‚Äî BullMQ internals exposed unauthenticated** ‚Äî `src/app/api/jobs/route.ts` ‚Äî 335-line route exposes full queue state, job payloads, cancel/retry management. No auth imports in file. Anyone can enumerate job payloads or cancel production jobs. *Unresolved for 8 weeks.*

6. **`/api/gamification` family ‚Äî XP, achievements, badges, challenges all writeable without auth** ‚Äî `src/app/api/gamification/route.ts`, `src/app/api/gamification/badges/route.ts`, `src/app/api/gamification/challenges/route.ts` ‚Äî Zero auth checks in all three files. Any unauthenticated actor can inflate XP or award achievements for any `userId` in the request body. *Unresolved for 8 weeks.*

7. **`/api/export` family ‚Äî export history and downloads accessible without auth** ‚Äî `src/app/api/export/history/route.ts`, `src/app/api/export/[id]/route.ts`, `src/app/api/export/[id]/download/route.ts`, `src/app/api/export/templates/route.ts` ‚Äî No auth in any of these four files. A caller with a known job UUID can download another user's export file. *Unresolved for 8 weeks.*

8. **`CLERK_CONFIGURED` dead reference bypasses auth in development** ‚Äî `src/lib/auth/api-key-context.ts:37-39` ‚Äî Post-Supabase migration, code still checks `process.env.NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`.
