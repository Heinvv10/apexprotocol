# Legacy Drizzle Migrations

These SQL files were hand-written directly (not generated by `drizzle-kit generate`) during the period when the drizzle-kit workflow was abandoned — roughly between migration 0010 and the Supabase migration on 2026-04-18.

They were applied manually to the production database but never registered in `drizzle/meta/_journal.json`. As of 2026-04-21 their schema changes were reconciled into the current `src/lib/db/schema/*.ts` and captured by the existing `0010_nosy_daimon_hellstrom` snapshot — running `drizzle-kit generate` against the live DB now produces "No schema changes, nothing to migrate".

Archived here on 2026-04-21 so `drizzle-kit` only sees the canonical, journaled migrations.

**Do not re-apply these files.** They are for historical reference only. The live DB already has every column/table/index they create.

If you need to reconstruct the prod schema from scratch (e.g. for a fresh environment), run the journaled migrations (`0000_*` through `0010_*`) via `drizzle-kit migrate`, then do `drizzle-kit introspect` against a known-good instance to pick up anything not captured by the canonical snapshots.
